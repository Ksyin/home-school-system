Firebase setup guide

Exact database used in this project: Cloud Firestore. File uploads use Firebase Storage. Login uses Firebase Authentication.

Back home

1. Create the Firebase project

Open Firebase Console and create a new project.
Inside the project, click Add app and choose Web.
Copy the web config object.
Paste it into assets/js/firebase-config.js.
export const firebaseConfig = {
  apiKey: "...",
  authDomain: "...firebaseapp.com",
  projectId: "...",
  storageBucket: "...firebasestorage.app",
  messagingSenderId: "...",
  appId: "..."
};

2. Enable Authentication

Go to Authentication → Sign-in method.
Enable Email/Password.
This system uses email/password for parent, tutor, and student logins.

3. Create Cloud Firestore

Go to Firestore Database and create the database.
Start in test mode first while setting up, then add the rules from the included docs/firestore.rules.txt.
Database type used here is Cloud Firestore, not Realtime Database.

4. Create Firebase Storage

Go to Storage and create the default bucket.
This stores worksheets, student submissions, portfolio files, and learning materials.

5. How login routing works

User signs in through Firebase Authentication.
System reads users/{uid} from Firestore.
Based on role, user is redirected automatically:
parent → /parent/dashboard.html
tutor → /tutor/dashboard.html
student → /student/dashboard.html

6. How learning materials work

Tutor is responsible for uploading learning materials.
Tutor uploads from the Tutor → Resources page.
Files go to Firebase Storage, then metadata is saved in Firestore resources.
Parents and students can view and download what has been shared.

7. How assignments work

Tutor creates an assignment in Tutor → Assignments.
Student sees it in Student → My Assignments.
Student uploads completed work in Student → Submit Work.
Submission file is stored in Firebase Storage, and a document is created in Firestore submissions.

8. If you want Realtime Database instead

This ZIP is built for Cloud Firestore, not Realtime Database.
Firestore is the better fit here because collections like assignments, submissions, reports, and portfolios are easier to model.
Realtime Database can work, but it becomes harder to query by role, student, tutor, status, and time.
For this kind of management system, Firestore is the recommended Firebase database.
